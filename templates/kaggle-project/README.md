## Workflow

1. Join the competition and download the data locally.
2. Make a baseline submission within the first day.
3. Track changes in small commits.
4. Move reusable logic out of notebooks and into `src/`.
5. End with a short case-study style write-up.

## Notes

- Keep large raw data local unless the license clearly allows publishing it.
- Prefer concise iteration notes over giant retrospective dumps.
